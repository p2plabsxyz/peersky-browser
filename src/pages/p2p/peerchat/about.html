About PeerChat

Small-group chat inside PeerSky Browser. You create a room, share a key, and everyone who has that key joins the same swarm and sees the same history; no chat server in the middle.
No account. Everything stays local on your machine (room list, profile, keys file); except what you explicitly sync over the peer network.
You own your chats: there’s no company holding logs or resetting your password; the room key is the shared secret.

What it does (technical)

  • Rooms with name, bio, optional picture; local profile.
  • One Hypercore feed per room; Hyperswarm for transport; live UI via SSE (receive-all).
  • Messages encrypted with AES-256-GCM derived from the room key before they’re stored in the feed.
  • Attachments go through a Hyperdrive (peerchat), listed in Settings → Archive like other PeerSky apps. No app-enforced upload size limit—only disk, memory while uploading, and the network.
  • Direct Messages (DMs): click a peer's avatar to send a private message; the recipient gets an accept/decline popup, and the room key is derived from both peer IDs so only those two people share it.
  • Optional Electron safeStorage for the local JSON that holds rooms and keys.

How the room key works

The key is a random secret shown as 64 hex characters. Anyone with it can join that room and read history from peers who have the data. Treat it like a shared password. Use Join room with the key; for room details or keys from your archive, see peerchat-rooms under Hyperdrives in Settings → Archive.
